# Отчёт о качестве данных

## 1. Введение

**Датасет:** FAANG Stock Market Data with Technical Indicators  
**Источник:** [Kaggle: vishardmehta/faang-stock-market-data-with-technical-indicators](https://www.kaggle.com/datasets/vishardmehta/faang-stock-market-data-with-technical-indicators)  
**Размер:** 14964 строки × 19 столбцов  

**Предметная область:**  
Финансовые рынки. Данные содержат исторические котировки (OHLCV) и рассчитанные технические индикаторы для 5 крупнейших технологических компаний США (Facebook/Meta, Apple, Amazon, Netflix, Google).

**Условный заказчик:**  
Частный трейдер, квант-аналитик (Quant Researcher) или инвестиционный фонд, разрабатывающий алгоритмические торговые стратегии.

**Возможные задачи ИАД:**
1. **Прогнозирование цены (Regression):** Предсказание цены закрытия на следующий день (`Next_Day_Close`) на основе технических индикаторов.
2. **Анализ волатильности:** Сравнение рисков (через `Volatility_7d` и `Bollinger Bands`) между разными тикерами.
3. **Классификация трендов:** Определение вероятности роста/падения цены на основе сигналов индикаторов (RSI, MACD).

## 2. Паспорт датасета

| Столбец | Тип (Pandas) | Описание | Примечания |
|---------|-----|----------|------------|
| Date | object | Дата торгового дня | Требует преобразования в datetime. |
| Ticker | object | Тикер компании (AAPL, AMZN, GOOGL, META, NFLX) | Категориальный признак. |
| Open, High, Low, Close | float64 | Цены открытия, максимума, минимума и закрытия | Основные рыночные данные. |
| Volume | int64 | Объем торгов | Может иметь экстремальные значения (выбросы) в дни отчетов. |
| SMA_7, SMA_21 | float64 | Скользящие средние (7 и 21 день) | Трендовые индикаторы. |
| EMA_12, EMA_26 | float64 | Экспоненциальные скользящие средние | Используются для MACD. |
| RSI_14 | float64 | Индекс относительной силы (14 дней) | Осциллятор (0-100). |
| MACD, MACD_Signal | float64 | Схождение/расхождение скользящих средних | Индикатор импульса. |
| Bollinger_Upper/Lower | float64 | Полосы Боллинджера | Индикаторы волатильности. |
| Daily_Return | float64 | Дневная доходность | Процентное изменение цены. |
| Volatility_7d | float64 | Волатильность за 7 дней | Мера риска. |
| Next_Day_Close | float64 | Цена закрытия следующего дня | Целевая переменная для предсказания. |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск | Предлагаемое решение |
|---|--------------|----------|-------------------|---------------------|
| 1 | Тип данных | Столбец `Date` имеет тип `string` | Невозможность корректной сортировки, фильтрации по времени и построения графиков. | Преобразовать в `datetime64[ns]` с помощью `pd.to_datetime`. |
| 2 | Выбросы | В столбце `Close` обнаружено 937 выбросов (6.26%) по методу IQR. | Модели линейной регрессии могут быть смещены. Однако для фин. данных это может быть нормой (кризисы, ралли). | Для фин. анализа: оставить как есть. Для простых ML моделей: использовать робастные методы или скейлинг (RobustScaler). |
| 3 | Мультиколлинеарность | Сильная корреляция между `Close`, `SMA`, `EMA`. | Переобучение линейных моделей, неустойчивые веса признаков. | Использовать регуляризацию (L1/L2) или отобрать подмножество признаков (например, только `Daily_Return` и осцилляторы). |
| 4 | Масштаб | Признаки имеют разный масштаб (Цена ~100-3000, RSI ~0-100, Volume ~10^6). | Плохая сходимость градиентных методов (нейросети, логистическая регрессия). | Нормализация (MinMax) или стандартизация (StandardScaler). |

*Примечание: Пропусков и полных дубликатов в данном наборе данных не обнаружено.*

## 4. План улучшения качества

**Приоритет 1 (критично):**  
Преобразование `Date` в формат даты-времени и установка его в качестве индекса для корректной работы с временными рядами.

**Приоритет 2:**  
Масштабирование данных. Разделение признаков на группы для разных моделей (например, цены отдельно, проценты отдельно) или применение `StandardScaler` ко всем числовым входам.

**Приоритет 3:**  
Инженерный анализ признаков. Создание лаговых переменных (Lag features) для учета автокорреляции, так как текущие индикаторы уже частично агрегируют историю.

## 5. Краткое резюме

Данные высокого качества: отсутствуют пропуски и технические дубликаты, что редкость для "сырых" финансовых данных. Основная работа потребуется на этапе предобработки (Feature Engineering): приведение типов, масштабирование и борьба с мультиколлинеарностью технических индикаторов. Выбросы, вероятно, являются естественной частью рыночной динамики и требуют осторожного обращения (не удалять слепо).
